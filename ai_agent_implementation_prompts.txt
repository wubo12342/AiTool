# AI Agent 小幫手實作指南 (提示詞集)

這份文件旨在幫助你將本專題中的「AI 智慧小幫手」升級為具備「Function Calling (Agent 工具調用)」能力的進階版本。你可以將以下提示詞輸入給 ChatGPT、Claude 或 Gemini 等 AI，引導它們幫你寫程式。

---

## 1. 專案背景資訊 (背景餵食)
在開始前，請先給 AI 這段背景，讓它了解你的環境：
「我正在開發一個 AI 工具導航平台，前端使用 Vue.js (Vite)，後端使用 PHP (XAMPP)，資料庫為 MySQL。目前我已經有一個基本的聊天視窗 UI。我希望實作 Function Calling 功能，讓 AI 小幫手能主動查詢資料庫中的『收藏列表』、『工具資料』與『用戶偏好』。請基於此架構協助我。」

---

## 2. 任務提示詞 A：定義 API 溝通協定
「請幫我設計一個 PHP 腳本 `chat_agent.php` 作為後端中轉站。功能要求如下：
1. 接收前端傳來的 JSON 請求（含用戶訊息）。
2. 使用 CURL 發送請求給 [OpenAI/Gemini] API。
3. 在請求中定義『Tools (Functions)』，例如：
   - `search_tools($keyword)`: 搜尋資料庫內容。
   - `get_user_favorites()`: 取得該用戶的收藏清單。
   - `get_user_prefs()`: 取得用戶的偏好設定。
4. 當 AI 回傳『工具呼叫請求 (Tool Calls)』時，PHP 要能識別並去執行對應的 SQL 查詢，最後將查詢結果再次傳回給 AI。」

---

## 3. 任務提示詞 B：寫出 Function Schema (工具說明書)
「請幫我寫出符合 [OpenAI/Gemini] 格式的 Function Schemas。我需要讓 AI 知道它有權限調用以下功能：
1. `search_database`: 搜尋我的工具分類與標籤（欄位包含 id, name, description, tags）。
2. `fetch_reviews`: 抓取特定工具的使用者評價。
3. `read_user_context`: 讀取該用戶在資料庫中存儲的職業背景與使用習慣。
請確保參數定義清晰，讓 AI 能精確判斷何時該調用。」

---

## 4. 任務提示詞 C：前端 Vue 互動邏輯 (處理遞迴呼叫)
「在我的 Vue 組件 `AiAssistant.vue` 中，我該如何處理 Agent 的對話循環？
1. 當 API 回傳的是普通文字，直接顯示。
2. 當 API 回傳的是工具呼叫（例如 AI 想查資料庫），後端 PHP 處理完後，前端是否需要重新發送請求？請幫我規劃這個流暢的對話循環邏輯，並加入 Loading 狀態讓使用者知道 AI 正在『查詢資料中』。」

---

## 5. 任務提示詞 D：提示詞工程 (System Prompt)
「請幫我寫一段給這個 Agent 小幫手的 System Prompt。它的身分是：
- 一個專業的 AI 工具顧問。
- 它知道它有權限存取資料庫，所以當用戶提到『我之前存過...』或『推薦給我適合我的...』時，它必須優先考慮使用工具去查詢，而不是瞎猜。
- 回答要親切且具備專業洞察力。」

---

## 💡 實作建議
在使用這些提示詞時，建議**分步驟進行**。先讓 AI 幫你寫出後端 PHP 的基本掛接，確認能成功連到 OpenAI/Gemini 後，再開始加入資料庫查詢的工具調用邏輯。
